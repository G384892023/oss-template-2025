# Parent (親コミット)

## 説明

新しいコミットを作成すると、それは「親」となる既存のコミットの履歴を継承します。
parentというのは親コミットすなわち１個前のコミットのハッシュ値です。
Gitのコミットオブジェクトは必ず1つ以上の親コミットを持っており、
親を順番にたどっていくことで履歴をさかのぼることができます。
複数のブランチで作業が進められた場合、マージ（結合）時に、異なるコミットが親となり、
それらのコミットを統合した新しいコミットが作られます。
この場合、統合前の複数のコミットが統合後のコミットの親になります。
